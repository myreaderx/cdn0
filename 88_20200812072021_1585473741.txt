Dock 上就是一种比什么都快的文件收集方式。</p><p>第一次使用 DEVONthink 3 时可能没自动呈现 Inbox，可以随着这个路径把它找出来：</p><pre class=\"language-\"><code>~/Library/Application Support/DEVONthink 3/Inbox\n</code></pre><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E6%8A%8A%20Inbox%20%E6%8B%96%E5%88%B0%20Dock%20%E4%B8%8A.png\" alt=\"\"><figcaption>把 Inbox 拖到 Dock 上</figcaption></figure><p>Inbox 也没什么理解门槛和使用成本，不管你是刚用 DEVONthink 搞不懂其他方式，还是明确需求确实不用全天候开着 DEVONthink（Inbox 里的文件要在 DEVONthink 打开时才会被自动吸进去），用 Inbox 完成日常资料收集都是最直观的。</p><p>至于前两种添加方式——Index 和 Import——更适合第一次使用 DEVONthink 时往里面大搬家，往后收集文件就看习惯先收集还是先整理，去选 Inbox 或 Sorter。诚然，刚开始用 DEVONthink 时理解资料迁移、添加的概念的确有点累，可一旦确定自己的文件使用流程，以后的收集操作就会轻车熟路。</p><h3>参考：专业级浏览和搜索</h3><p>资料收集不是一蹴而就，往往以前的文件也会派上用场。我没有那么依赖 Sorter、更喜欢简单的 Inbox，一部分原因也是自己已经做了很久的整理工作，不是非常需要来一个文件就整理一下。</p><p>总之，有了一定数量的素材后就可以开始组织、使用它们，发挥 DEVONthink 资料库的参考作用。DEVONthink 3 在文件浏览形式和搜索方式上都比 Finder 自由，在它里面看参考资料更舒服。</p><p>打开 DEVONthink，最让人印象深刻的便是其文件浏览方式——<strong>左侧文件结构层层递进，中间文件预览尽收眼底，右侧文件属性一应俱全</strong>。新手可能要犯恐高症，不过已经大家既然已经不满于 Finder，相信一定程度上就是冲着这个界面来的。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-DEVONthink%20%E4%BF%A1%E6%81%AF%E9%BD%90%E5%85%A8%E7%9A%84%E7%95%8C%E9%9D%A2.png\" alt=\"\"><figcaption>DEVONthink 信息齐全的界面</figcaption></figure><p>左侧一栏名为 Sidebar（侧边栏），中间是文件 Preview（预览区），右边则是 Inspectors（检视窗）。有时可能不慎到快捷键把左右两栏隐藏起来，可以照着名字在上方工具栏里按图索骥，把这两个强大文件浏览界面找回来。</p><p>一个文件在哪个位置、内容是什么、有哪些属性、打上了什么标签……关于这个文件的几乎所有信息，都在同一个界面里中一览无遗。DEVONthink 里还有很多可选的浏览设置，但有这左中右三大块兜底，已经比原生邮件那一套三栏布局更加周全，遑论简洁为主的 Finder。</p><p>在这个界面下浏览文件已经是一种享受，仿佛升了一个维度而往下俯拾昔日的文件。不过，我们还要解决另一个大问题：<strong>文件搜索。</strong></p><p>好消息是，Finder 里的搜索语法和智能文件夹技巧几乎可以照搬到 DEVONthink 里；而且 DEVONthink 里面没有各种系统文件的干扰，我们的搜索结果会更干净。</p><p>我已经知道，常用搜索条件可以保存成<strong>智能文件夹</strong>，DEVONthink 3 也继承了这一搜索功能（上一代条件不太全，建议使用 DEVONthink 3）。我有两个智能文件夹，它们可以根据文章的 URL 自动过滤出 Power+ 1.0 和 Power+ 2.0 的文章，想引用自己和同事的作品比较方便。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E6%B1%87%E9%9B%86%20Power-%20%E6%96%87%E7%AB%A0%E7%9A%84%E6%99%BA%E8%83%BD%E6%96%87%E4%BB%B6%E5%A4%B9.png\" alt=\"\"><figcaption>汇集 Power+ 文章的智能文件夹</figcaption></figure><p>除了不掺系统文件夹等「杂质」，DEVONthink 的智能文件夹也有比较大的自动化处理空间，可以<strong>对里面的文件进行批量操作</strong>，这在 Finder 和 Automator 那边是很难实现的。一个例子是处理扫描版 PDF，先把把它们过滤出来，再对搜到的文件进行 OCR 字符识别，每次有新 PDF 进来就处理一份，以后打开 DEVONthink 就能直接搜所有扫描件的内容。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E8%87%AA%E5%8A%A8%E5%A4%84%E7%90%86%E7%9A%84%E8%A7%84%E5%88%99.png\" alt=\"\"><figcaption>自动处理的规则</figcaption></figure><p>有点遗憾的是这个「智能文件夹」将和自动化规则绑定，不会单独出现，增加了大家理解的成本。如果以后可以直接对一个现成智能文件夹进行自动化操作，会更加容易理解和使用（目前似乎只有 Hazel 可以直接处理 Finder 智能文件夹中的内容）。</p><p>至于偶尔一用的搜索，可以试试<strong>人工智能引擎</strong>。早先接触到所谓 AI 引擎时我很不屑，部分原因是这东西准确度有限，在中文下更惨；部分原因是我的多数搜索已经靠高级搜索语法和条件实现了，似乎没有 AI 的出场余地。</p><p>实际上，有些搜索条件可能比较模糊或笼统，比如 <code>任务管理</code> 相关的文章，有些文章里甚至不会出现这四个字，这时候 DEVONthink 的 AI 建议就可以充当搜索的辅助；刚好今年 DEVONthink 3 多少也改进了中文搜索，我慢慢开始通过 AI 引擎找<strong>相关素材</strong>。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E9%80%9A%E8%BF%87%20DEVONthink%20%E6%89%BE%E7%9B%B8%E5%85%B3%E7%B4%A0%E6%9D%90.png\" alt=\"\"><figcaption>通过 DEVONthink 找相关素材</figcaption></figure><p>所谓找「相关」素材，主要有两层含义：</p><ol><li>搜一个关键词，在搜索结果中列出相关的文件</li><li>选中一个文件，在 Inspectors 检视区域中举出同类的文件（即老用户熟悉的 See Also &amp; Classify 功能）</li></ol><p>上图预览区和检视区很多红彤彤的进度条就表示两种相关度的高低，目前还是在英文下更准一点，中文文件估计要靠打自己标签、加注释来提高权重，从而增强 AI 建议的准确性。很多时候我们可能想不起一个关键词，或者不知道怎么描述它，AI 建议或多或少能够帮上忙；这让我想起 iTunes 里有个推荐相似歌曲的 <a href=\"https://www.apple.com/legal/internet-services/itunes/us/genius.html\">Genius</a> 功能，和现在的 Spotify 很一样都是靠机器和算法给出建议。在 DEVONthink 资料库日益扩大的过程中，Inspectors 是一个减轻思考负担的好工具。</p><p>不论如何，在 DEVONthink 里我们都更容易中找到自己需要的文件，也可以通过更加专业的方式来浏览它们。把素材库搬进 DEVONthink 后，一个明显感受就是可以专注于查看这些参考资料，少花一些时间在整理和翻找上。</p><h3>成品：保存完整网页</h3><p>写作本身是不会在 DEVONthink 里进行的，我的工作常态是开着 Ulysses 全屏写文章，后台运行 DEVONthink 随时供参考。文章发布以后，再次轮到 DEVONthink 出场：<strong>给成品文章存档。</strong></p><p>说到网页存档，文件格式的选择是很让人犯难的。常见如 PDF 打印、页面抓取、HTML 文件保存等等方式都让人感觉缺了点啥，PDF 无法保存动图和视频，抓下来的页面不能保留页面原样式……其实在这些抱怨背后，可以总结出我们对网页存档的两点需求：</p><ol><li>最基础的，<strong>要尽可能尊重页面原始样貌</strong>，原有的元素和版式尽量保存下来。不管用什么工具（其实 DEVONthink 以外的工具基本做不到），都要保证网页资料是「第一手」的，不能像在笔记软件里那样裁裁剪剪过。</li><li>进阶一点，<strong>要能够导出 PDF、HTML 这些常规格式</strong>，方便分享给别人。私人资料库里的文件格式冷僻一点没关系，但是要能转换成常见格式，不然就太依赖资料库工具，灵活性和实用性会打折扣。</li></ol><p>试过各种所谓「信息管理工具」后，我找到最保险的方式是 <strong>使用 DEVONthink 把网页存为 Web Archive 原始存档</strong>。这种格式被 macOS 原生支持，在 Safari 缓存中也有用到，不管图文还是视频、甚至连离线小游戏都能保存下来。</p><p>DEVONthink 本身就可以把免费网页保存为 Web Archive，此外，一些浏览器自己也可以生成 Web Archive 文件，两种方法可以组合起来进行网页收集，基本上没什么死角：</p><p><strong>电脑上</strong>：无需打开 DEVONthink，直接在 Safari 中按下 <code>⌘Command-S</code> 保存网页源文件到 Inbox，下次开启 DEVONthink 时就会自动导入网页存档。在浏览器里存 Web Archive 的首要原因是<strong>速度快</strong>，直接快捷键完成操作；还有一个原因是只要网页已经加载好，<strong>付费内容也可以保存</strong><sup class=\"ss-footnote\" href=\"\" title=\"通过 DEVONthink 插件是抓不到的。\" footnote-id=\"2\">2</sup>，我自己就通过 Web Archive 来收集 Power+ 文章和付费教程。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E5%9C%A8%20Safari%20%E9%87%8C%E7%9B%B4%E6%8E%A5%E5%AD%98%E7%BD%91%E9%A1%B5.png\" alt=\"\"><figcaption>在 Safari 里直接存网页</figcaption></figure><p><strong>iOS 设备上</strong>：手机和平板上可以通过分享菜单把网页 Clip（剪藏）到 DEVONthink，它会自动存一份 Web Archive 文件。从尊重原网页的角度看，剪藏样式要避免「clutter free」<sup class=\"ss-footnote\" href=\"\" title=\"类似去广告的重排版效果。\" footnote-id=\"3\">3</sup>，否则搞不好就把有用的内容当广告裁掉了。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E4%BD%BF%E7%94%A8%20DEVONthink%20To%20Go%20%E6%94%B6%E9%9B%86%E7%BD%91%E9%A1%B5.PNG\" alt=\"\"><figcaption>使用 DEVONthink To Go 收集网页</figcaption></figure><p>如果你已经升级了 iOS 13，那在 iPad 上也可以直接按快捷键 <code>⌘Command-S</code>（和电脑端如出一辙）来保存网页。当然，保存位置可以选择 DEVONthink 文件夹。iPhone 上则可以在通过分享菜单导出网页前选择格式，选中「网页归档」来导出 Web Archive。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-iPad%20%E7%89%88%20Safari%20%E5%8F%AF%E4%BB%A5%E7%9B%B4%E6%8E%A5%E4%BF%9D%E5%AD%98%20webarchive%20%E6%96%87%E4%BB%B6.PNG\" alt=\"\"><figcaption>iPad 版 Safari 可以直接保存 webarchive 文件</figcaption></figure><p>存 Web Archive 有点像全盘备份，你越是尊重元数据，它们越是可能在意想不到的地方帮上忙。有一次我的博客配图因为图床问题在一夜之间全部不能显示，幸好我每发一篇就在 DEVONthink 里存一份 Web Archive，成品和读者评论都能找回来。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E9%80%9A%E8%BF%87%20Web%20Archive%20%E8%BF%9E%E6%96%87%E7%AB%A0%E5%B8%A6%E8%AF%84%E8%AE%BA%E4%B8%80%E8%B5%B7%E5%AD%98%E4%B8%8B%E6%9D%A5.png\" alt=\"\"><figcaption>通过 Web Archive 连文章带评论一起存下来</figcaption></figure><p>存好 Web Archive 事情还没完，万一要发给别人总免不了转成常见格式。这事儿搁 Finder 估计又跳得到 Safari、再用虚拟打印机打一份 PDF，不过在 DEVONthink 里已经集成了格式转换功能，直接右键 Web Archive 文档就能看到转换成 PDF 的选项，转出来的样式相对朴素。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E7%94%A8%20DEVONthink%20%E7%9B%B4%E6%8E%A5%E6%8A%8A%E7%BD%91%E9%A1%B5%E8%BD%AC%E6%88%90%20PDF.png\" alt=\"\"><figcaption>用 DEVONthink 直接把网页转成 PDF</figcaption></figure><p>有些时候给别人分享的文章比较笼统，比如有人问我推荐 AppleScript 的参考资料，我便需要就这一主题整理一份<strong>阅读清单</strong>给他，这在 DEVONthink 里可以借助文件元数据和脚本来实现。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com/ee/2020-08-12-%E4%BB%8E%20DEVONthink%20%E5%AF%BC%E5%87%BA%E9%98%85%E8%AF%BB%E6%B8%85%E5%8D%95.png\" alt=\"\"><figcaption>从 DEVONthink 导出阅读清单</figcaption></figure><p><a href=\"https://cdn.sspai.com/ee/2020-08-12-Export%20Markdown%20Links.scpt.zip\"><strong>&gt; 动作下载 🔗</strong></a></p><p>这个动作可以把当前在 DEVONthink 里选中的文件（这里指的是 Web Archive）导出为 Markdown 链接列表、再拷贝到剪贴板，方便分享给别人。之前我通过 Pocket 创建阅读列表，而 DEVONthink 的元数据更丰富一些，理出来的文章相关度会更高（还可以手动剔除不想要的），适合对方需要精度的场景。</p><p>话说回来，用 Web Archive 格式的人仍然不多，和 DEVONthink 一组合，估计就更小众，所以最后再简单解答一些相关疑惑。大家最关心的可能是阅读体验，Web Archive 的确不如稍后读工具或剪藏工具那么讨好眼球，但是后两者抓下来的页面往往连原始信息都有丢出，作为资料库只能说不合格；至于和 PDF 的取舍，如果考虑<strong>跨平台</strong>因素的话还是 DEVONthink&amp;Web Archive 的组合优势更明显：移动端的 DEVONthink To Go 可以查看 Web Archive 文件，而且在 iPhone 上还可以自动适应屏幕尺寸（毕竟是网页嘛），浏览起来实际上比 PDF 还方便。</p><p>当写作成为我的主要工作后，把相关文件从 Finder 剥离出来放进 DEVONthink 统一管理几乎是理所当然的选择。创建并维护一个专用资料库后，写作时得以更加专注于写作，而不用被一些来路不明的文件打搅。</p><h2>小结</h2><p>在写这一章前我和 @Hum 沟通了很久，因为我一直感觉很多读者对 DEVONthink 存在误解，把它视作了难以驯服的烈马，于是我就想写一篇文章来终结这种「迷信」。但假如真这么写了，文章的节奏就会变得很怪异，即便不引发争论，最理想的情况下也会在前半章就搞得大家对 DEVONthink 胃口全无。这对于真心想用 DEVONthink 的读者来说是不公平的。</p><p>所以最后在 @Hum 的梳理和帮助下，文章跟随全教程的步调，衔接前一章的内容、从 Finder 的潜在问题慢慢引出 DEVONthink，也许显得慢热，但应该是介绍这款话题性工具最合适的方式了。</p><p>DEVONthink 很强大，但它不适合所有人。如果一上来——特别是在没有把 Finder 吃透的情况下——就直接跳进 DEVONthink 的体系，错过许多原本就免费的功能不说，还很容易被 DEVONthink 的复杂界面搞糊涂，变成「为了用而用」。</p><p>希望通过这一章的阅读，首先能帮大家搞明白自己到底需不需要 DEVONthink，在此基础上再给更专精的文件管理留下一个引子。</p><p>© 本文著作权归作者所有，并授权少数派独家使用，未经少数派许可，不得转载使用。</p>","descriptionType":"text/html","publishedDate":"Wed, 12 Aug
