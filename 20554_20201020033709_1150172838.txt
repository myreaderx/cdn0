Historically, an Android view hierarchy has been represented as a tree of UI widgets. As the state of the app changes, the UI hierarchy needs to be updated to display the current data. The most common way of updating the UI is to walk the tree using functions like <a href=\"https://android.devsite.corp.google.com/reference/android/view/View#findViewById(int)\">findViewById()</a>, and change nodes by calling methods like: <pre> button.setText(String) </pre> <pre>container.addView(View) </pre> <pre> img.setImageBitmap(Bitmap) </pre> These methods change the internal state of the widget. Not only can this be tedious, but updating views manually increases the likelihood of errors (e.g. forgetting to update a view).\n</p>\n<p>\nJetpack Compose is a fully declarative component-based approach, meaning you describe your UI as functions that transform data into a UI hierarchy. When the underlying data changes, the Compose framework automatically updates the UI hierarchy for you, making it simple to build UIs easily and quickly.\n</p>\n<p>\n\n<center> <iframe width=\"560\" height=\"315\" src=\"https://www.youtube.com/embed/SMOhl9RK0BA\" frameborder=\"0\" allow=\"accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture\" allowfullscreen></iframe></p> </center>\n\n\n<h2>Full interop with existing Android views</h2>\n\n\n<p>\nAdopting any new framework is a big change for existing projects and codebases, which is why we’ve designed Compose to be as easy to adopt as Kotlin — it is fully interoperable with existing Android code, from day one. \n</p>\n<p>\nMigrating to Compose depends on you and your team. If you're building a new app, the best option might be to implement your entire UI with Compose. We know that most of you have large existing codebases, so rather than rewriting your app, you can combine Compose with your existing UI design.\n</p>\n<p>\nThere are two main ways you can combine Compose with a view-based UI:\n</p>\n<ul>\n\n<li>You can add Compose elements into your existing UI, either by creating an entirely new Compose-based screen, or by adding Compose elements into an existing fragment or view layout.\n\n<li>You can add a view-based UI element into your composable functions. Doing so lets you add non-Compose widgets, such as MapView or WebView, into a Compose-based design.\n</li>\n</ul>\n<p>\nWe’ve also published a new library, <a href=\"https://material-components.github.io/material-components-android-compose-theme-adapter/\">MDC Compose Theme Adapter</a>, which allows you to reuse your existing <a href=\"https://material.io/develop/android/\">Material Components</a> themes within your Compose UI.\n</p>\n<p>\nTo learn more, try the <a href=\"https://codelabs.developers.google.com/codelabs/android-compose-migration\">Compose for existing apps codelab</a> or check out these two samples:\n</p>\n<ul>\n\n<li><a href=\"http://tivi.app\">Tivi</a> and <a href=\"http://goo.gle/sunflower-compose\">Sunflower</a> are existing apps which are being integrated with Compose\n\n<li>Crane <a href=\"https://github.com/android/compose-samples/\">sample app</a>, embeds a MapView in Compose\n</li>\n</ul>\n\n<center> <iframe width=\"560\" height=\"315\" src=\"https://www.youtube.com/embed/PjQdFmiDgwk\" frameborder=\"0\" allow=\"accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture\" allowfullscreen></iframe> </center>\n\n<h2>Powerful Tools</h2>\n\n\n<p>\nJetpack Compose is built with powerful tooling in Android Studio, designed to help you iterate quickly on the piece of UI you’re working on.\n</p>\n<p>\nThe Compose layout preview enables you to preview your Compose components without having to deploy your app to a device or emulator.
