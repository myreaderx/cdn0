0);line-height: inherit;border-radius: 4px;background: rgb(248, 248, 248);\">分布式设计</code>，这种设计的特点如下</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 个不同的组织管理。根域名服务器的清单和组织机构可以在 https://root-servers.org/ 中找到，根域名服务器提供 TLD 服务器的 IP 地址。</p></li><li style=\"margin-bottom: 0.5em;font-size: inherit;color: inherit;line-height: inherit;\"><p><code 服务器</code>，对于每个顶级域名比如 com、org、net、edu 和 gov 和所有的国家级域名 uk、fr、ca 和 jp 都有 TLD 服务器或服务器集群。所有的顶级域列表参见 https://tld-list.com/ 。TDL 服务器提供了权威 DNS 服务器的 IP 地址。</p></li><li style=\"margin-bottom: 0.5em;font-size: 4px;background: rgb(248, 248, 248);\">权威 DNS 服务器</code>，在因特网上具有公共可访问的主机，如 Web 服务器和邮件服务器，这些主机的组织机构必须提供可供访问的 DNS 记录，这些记录将这些主机的名字映射为 IP 地址。一个组织机构的权威 DNS 服务器收藏了这些 DNS 记录。</p></li></ul><p 服务器的设计理念之后，我们回到 DNS 查找的步骤上来，DNS 的查询方式主要分为三种</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb 2px;padding: 2px 4px;font-size: inherit;color: rgb(233, 105, 0);line-height: inherit;border-radius: 4px;background: rgb(248, 248, 248);\">非递归查询</code>：当 DNS 解析器客户端查询 DNS 服务器以获取其有权访问的记录时通常会进行此查询，因为其对该记录具有权威性，或者该记录存在于其缓存内。DNS 服务器通常会缓存 DNS 记录，查询到来后能够直接返回缓存结果，以防止更多带宽消耗和上游服务器上的负载。</p></li></ul><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, inherit;\">DNS 解析器</strong>，它一般是 ISP 维护的 DNS 服务器，它的主要职责就是通过向网络中其他 DNS 服务器询问正确的 IP 地址。</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: 顶级域名服务器会将其重定向到google.com 权威服务器。google.com 名称服务器将在其 DNS 记录中找到 maps.google.com 匹配的 IP 地址，并将其返回给您的 DNS 解析器，然后将其发送回你的浏览器。</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: normal;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal><p style=\"font-size: inherit;color: inherit;line-height: inherit;\">什么时候需要发送 ARP 请求呢？</p></blockquote><p 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>ARP 协议的全称是 <code style=\"margin-right: 2px;margin-left: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>简而言之，ARP 就是一种解决地址问题的协议，它以 IP 地址为线索，定位下一个应该接收数据分包的主机 MAC 地址。如果目标主机不在同一个链路上，那么会查找下一跳路由器的 MAC 地址。</p><blockquote style=\"padding: 15px 15px 15px 1rem;color: rgb(102, 102, 102);border-left-width: 6px;border-left-color: rgb(33, 152, 99);font-size: style=\"font-size: inherit;color: inherit;line-height: inherit;\">关于为什么有了 IP 地址，还要有 MAC 地址概述可以参看知乎这个回答 https://www.zhihu.com/question/21546408</p></blockquote><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>ARP 的大致工作流程如下</p><p 位于同一链路，不需要经过路由器的转换，主机 A 向主机 B 发送一个 IP 分组，主机 A 的地址是 192.168.1.2 ，主机 B 的地址是 192.168.1.3，它们都不知道对方的 MAC 地址是啥，主机 C 和 主机 D 是同一链路的其他主机。</p><figure style=\"font-size: 16px;color: rgb(62, 62, 62);line-height: 的 MAC 地址，通过主机 A 会通过<code style=\"margin-right: 2px;margin-left: 2px;padding: 2px 4px;font-size: inherit;color: rgb(233, 105, 0);line-height: inherit;border-radius: 4px;background: rgb(248, 248, inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>这里会涉及到 <strong style=\"color: rgb(19, 92, 224);font-size: inherit;line-height: inherit;\">ARP 缓存</strong>的概念。</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>现在你知道了发送一次 IP 分组前通过发送一次 ARP 请求就能够确定 MAC 地址。那么是不是每发送一次都得经过广播 -&gt; 封装 ARP 响应 -&gt; 返回给主机这一系列流程呢？</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>想想看，浏览器是如何做的？浏览器内置了缓存能够缓存你最近经常使用的地址，那么 ARP 也是一样的。ARP 高效运行的关键就是维护每个主机和路由器上的 <code style=\"margin-right: 2px;margin-left: 2px;padding: 2px 4px;font-size: inherit;color: rgb(233, 105, 0);line-height: inherit;border-radius: 4px;background: rgb(248, 248, 248);\">ARP 缓存(或表)</code>。这个缓存维护着每个 IP 到 MAC 地址的映射关系。通过把第一次 ARP 获取到的 MAC 地址作为 IP 对 MAC 的映射关系到一个 ARP 缓存表中，下一次再向这个地址发送数据报时就不再需要重新发送 ARP 请求了，而是直接使用这个缓存表中的 MAC 地址进行数据报的发送。每发送一次 ARP 请求，缓存表中对应的映射关系都会被清除。</p><p style=\"margin-top: 1.5em;margin-bottom: 1.5em;font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb>通过 ARP 缓存，降低了网络流量的使用，在一定程度上防止了 ARP 的大量广播。</p><figure style=\"font-size: 16px;color: rgb(62, 62, 62);line-height: inherit;font-family: \" helvetica neue sans gb yahei arial sans-serif start normal rgb><img
