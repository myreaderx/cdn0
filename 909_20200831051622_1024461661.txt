id=\"%E7%94%9F%E4%BA%A7%E8%BF%87%E7%A8%8B%E4%B8%AD%E5%AF%BC%E8%87%B4%E9%94%99%E8%AF%AF%E7%9A%84%E4%B8%A4%E7%A7%8D%E8%A1%8C%E4%B8%BA\" class=\"mume-header\">生产过程中导致错误的两种行为</h3>\n<p>这是考虑容错系统的另一种方法。当你在本地运行应用程序服务时，每件事似乎都很完美。棒极了！但当你提升服务到生产环境时，一切都会变得一团糟。在这种情况下，容错系统通过解决两个问题来提供帮助：故障停止行为和拜占庭行为。</p>\n<h4 id=\"%E6%95%85%E9%9A%9C%E5%81%9C%E6%AD%A2%E8%A1%8C%E4%B8%BA\" class=\"mume-header\">故障停止行为</h4>\n<p>故障停止行为是运行中系统突然停止运行或者系统中的某些部分发生了故障。服务器停机时间和数据库不可访问都属于此种类型。举个例子，在下图中，由于服务 2 无法访问，因此服务 1 无法与服务 2 进行通信。</p>\n<p><img title=\"Fail-stop behavior due to Service 2 downtime\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205112y5ylcyculncqq7o3.jpg\" alt=\"服务 2 停机导致的故障停止行为\" /></p>\n<p>但是，如果服务之间存在网络问题，也会出现此问题，如下图所示：</p>\n<p><img title=\"Fail-stop behavior due to network failure\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205113z4ykg4grysegkob3.jpg\" alt=\"网络故障导致的故障停止行为\" /></p>\n<h4 id=\"%E6%8B%9C%E5%8D%A0%E5%BA%AD%E8%A1%8C%E4%B8%BA\" class=\"mume-header\">拜占庭行为</h4>\n<p>拜占庭行为是指系统在持续运行，但并没有产生预期行为（例如：错误的数据或者无效的数据）。</p>\n<p>如果服务 2 的数据（值）已损坏则可能会发生拜占庭故障，即使服务看起来运行得很好，比如下面的例子:</p>\n<p><img title=\"Byzantine failure due to corrupted service\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205113qg96xllkqkxk7xcz.jpg\" alt=\"因服务损坏而导致的拜占庭故障\" /></p>\n<p>或者，可能存在恶意的中间人在服务之间进行拦截，并注入了不需要的数据：</p>\n<p><img title=\"Byzantine failure due to malicious middleman\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205117gnzppmt6p7iq4lzu.jpg\" alt=\"恶意中间人导致的拜占庭故障\" /></p>\n<p>无论是故障停止和拜占庭行为，都不是理想的情况，因此我们需要一些预防或修复它们的手段。这里容错系统就起作用了。以下是可以帮助你解决这些问题的 8 个开源工具。</p>\n<h3 id=\"%E6%9E%84%E5%BB%BA%E5%AE%B9%E9%94%99%E7%B3%BB%E7%BB%9F%E7%9A%84%E5%B7%A5%E5%85%B7\" class=\"mume-header\">构建容错系统的工具</h3>\n<p>尽管构建一个真正实用的容错系统涉及到深入的&#8220;分布式计算理论&#8221;和复杂的计算机科学原理，但有许多的软件工具（其中许多是开源软件）通过构建容错系统来减轻不良后果的影响。</p>\n<h4 id=\"%E6%96%AD%E8%B7%AF%E6%A8%A1%E5%BC%8Fhystrix-%E5%92%8C-resilience4j\" class=\"mume-header\">断路模式：Hystrix 和 Resilience4j</h4>\n<p><a href=\"https://martinfowler.com/bliki/CircuitBreaker.html\">断路模式</a>是一种技术，它有助于在服务失败时返回准备好的虚拟回应或者简单回应。</p>\n<p><img title=\"Circuit breaker pattern\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205120uiw947fgfula94pa.jpg\" alt=\"断路模式\" width=\"650\" height=\"407\" /></p>\n<p>奈飞开源的&#160;<a href=\"https://github.com/Netflix/Hystrix/wiki\">Hystrix</a>&#160;是断路模式中最流行的应用。</p>\n<p>我之前工作过的很多家公司都在用这款出色的工具。令人意外的是，奈飞宣布将不再更新 Hystrix（是的，我知道了）。相反，奈飞建议使用另一种支持 Java 8 和函数式编程的&#160;<a href=\"https://github.com/resilience4j/resilience4j\">Resilence4j</a>&#160;之类的替代解决方案，或者类似于&#160;<a href=\"https://medium.com/@NetflixTechBlog/performance-under-load-3e6fa9a60581\">Adaptive Concurrency Limit</a>&#160;的替代解决方案。</p>\n<h4 id=\"%E8%B4%9F%E8%BD%BD%E5%9D%87%E8%A1%A1nginx-%E5%92%8C-haproxy\" class=\"mume-header\">负载均衡：Nginx 和 HaProxy</h4>\n<p>负载均衡是分布式系统中最基本的概念之一，要想拥有一个生产质量的环境，必须有负载均衡的存在。要理解负载均衡器，首先我们需要明白冗余的概念。每个生产级的 Web 服务都有多个服务器在某个服务器宕机时提供冗余来接管和维持服务。</p>\n<p><img title=\"Load balancer\" src=\"https://img.linux.net.cn/data/attachment/album/202008/30/205124jv01giikl55v58yi.jpg\" alt=\"负载均衡\" width=\"650\" height=\"411\" /></p>\n<p>想想现代飞机：它们的双引擎提供冗余，使它们即使在一个引擎着火的情况下也能安全的着陆。（这也有助于大多数商用飞机拥有最先进的自动化系统）。但是，拥有多引擎（或者多服务器）也意味着必须存在一些调度机制在故障发生时有效地对系统进行路由。</p>\n<p>负载均衡器是一种通过平衡多个服务节点来优化大流量事务的设备或者软件。举个例子，当数以千计的请求涌入时，负载均衡器可以作为中间层在不同的服务器间进行路由和平均分配流量。如果一台服务器宕机，负载均衡器会将请求转发给其它运行良好的服务器。</p>\n<p>有许多可用的负载均衡器，但其中最出名的两个就是 Nginx 和 HaProxy。</p>\n<p><a href=\"https://www.nginx.com\">Nginx</a>&#160;不仅仅是一个负载均衡器，它还是 HTTP 和反向代理服务器、邮件代理服务器和通用 TCP/UDP 代理服务器。Groupon、Capital One、Adobe 和 NASA 等公司都在使用它。</p>\n<p><a href=\"https://www.haproxy.org\">HaProxy</a>&#160;也很受欢迎，因为它是一个免费的、非常快且可靠的解决方案，它为基于 TCP 和 HTTP 的应用程序提供高可用性、负载平衡和代理。许多大型网络公司，包括 Github、Reddit、Twitter 和 Stack Overflow 都使用 HaProxy。是的，Red Hat Enterprise Linux 同样支持 HaProxy 设置。</p>\n<h4 id=\"%E5%8F%82%E4%B8%8E%E8%80%85%E6%A8%A1%E5%9E%8Bakka\" class=\"mume-header\">参与者模型：Akka</h4>\n<p><a href=\"https://en.wikipedia.org/wiki/Actor_model\">参与者模型</a>是一种并发设计模式，当作为基本计算单位的&#8220;参与者&#8221;接收到消息时，它会分派责任。一个参与者可以创建更多的参与者，并将消息委派给他们。</p>\n<p><a href=\"https://akka.io\">Akka</a>&#160;是最著名的参与者模型实现之一。该框架同时支持基于 JVM 的 Java 和 Scala。</p>\n<h4 id=\"%E4%BD%BF%E7%94%A8%E6%B6%88%E6%81%AF%E9%98%9F%E5%88%97%E7%9A%84%E5%BC%82%E6%AD%A5-%E9%9D%9E%E9%98%BB%E5%A1%9E-iokafka-%E5%92%8C-rabbitmq\" class=\"mume-header\">使用消息队列的异步、非阻塞 I/O：Kafka 和 RabbitMQ</h4>\n<p>多线程开发在过去很流行，但是现在已经不鼓励这种做法了，取而代之的是异步的、非阻塞的 I/O 模式。对于 Java，这一点在&#160;<a href=\"https://jcp.org/aboutJava/communityprocess/final/jsr220/index.html\">EnterpriseJavaBean（EJB）规范</a>中得到了明确的规定：</p>\n<blockquote>\n<p>&#8220;企业 bean 一定不能使用线程同步原语来同步多个实例的执行。&#8221;</p>\n<p>&#8220;企业 bean 不得试图去管理线程。企业 bean 不得试图启动、停止、挂起或恢复线程，或者去更改线程的优先级或者名称。企业 bean 不得试图管理线程组。&#8221;</p>\n</blockquote>\n<p>如今，虽然还有其他做法，如流 API 和参与者模型，但像&#160;<a href=\"https://kafka.apache.org\">Kafka</a>&#160;和<a href=\"https://www.rabbitmq.com\">RabbitMQ</a>&#160;之类的消息队列为异步和非阻塞 I/O 功能提供了开箱即用的支持，同时它们也是功能强大的开源工具，通过处理并发进程可以替代线程。</p>\n<h4 id=\"%E5%85%B6%E4%BB%96%E7%9A%84%E9%80%89%E6%8B%A9eureka-%E5%92%8C-chaos-monkey\" class=\"mume-header\">其他的选择：Eureka 和 Chaos Monkey</h4>\n<p>用于容错系统其它有用的工具包括奈飞的&#160;<a href=\"https://github.com/Netflix/eureka\">Eureka</a>&#160;之类的监控工具，以及像&#160;<a href=\"https://github.com/Netflix/chaosmonkey\">Chaos Monkey</a>&#160;这样的压力测试工具。它们旨在通过在较低环境中的测试，如集成（INT）、质量保障（QA）和用户接受测试（UAT）来早早发现潜在问题以防止在转移到生产环境之前出现潜在问题。</p>\n<p>你在使用什么开源工具来构建一个容错系统呢？请在评论中分享你的最爱。</p>\n<hr />\n<p>via:&#160;<a href=\"https://itsfoss.com/linux-system-monitoring-tools/\">命令行进程监控工具</a>是&#160;<code>top</code>&#160;和它那色彩斑斓、功能丰富的表弟&#160;<a href=\"https://hisham.hm/htop/\">htop</a>。</p>\n<p>要<a href=\"https://itsfoss.com/monitor-cpu-gpu-temp-linux/\">监控 Linux 上的温度</a>，可以使用&#160;<a href=\"https://github.com/lm-sensors/lm-sensors\">lm-sensors</a>。同样，还有很多实用工具可以监控其他实时指标，如磁盘 I/O、网络统计等。</p>\n<p><a href=\"https://nicolargo.github.io/glances/\">Glances</a>&#160;是一个系统监控工具，它把这些都联系在一起，并提供了更多的功能。我最喜欢的是，你可以在远程 Linux 服务器上运行 Glances 来监控本地系统的系统资源，也可以通过 Web 浏览器监控。</p>\n<p>下面是它的外观。下面截图中的终端已经<a href=\"https://itsfoss.com/pywal/\">用 Pywal 工具美化过，可以根据壁纸自动改变颜色</a>。</p>\n<p><img src=\"https://img.linux.net.cn/data/attachment/album/202008/30/212845k997iv9vzu88hlnl.png\" alt=\"\" /></p>\n<p>你也可以将它集成到像&#160;<a href=\"https://grafana.com/\">Grafana</a>&#160;这样的工具中，在一个直观的仪表盘中监控统计数据。</p>\n<p>它是用 Python 编写的，这意味着它的绝大多数功能都可以在大多数平台上使用。</p>\n<h3 Data In Grafana Dashboard\" /></p>\n<p>让我们快速浏览一下 Glances 提供的主要功能：</p>\n<ul>\n<li>可以监控系统上的 15 个之多的指标（包括 Docker 容器）。</li>\n<li>灵活的使用模式：单机模式、客户端-服务器模式、通过 SSH 和 Web 模式。</li>\n<li>可用于集成的各种 REST API 和 XML-RPC API。</li>\n<li>支持将数据轻松导出到不同的服务和数据库。</li>\n<li>高度的可配置性和适应不同的需求。</li>\n<li>非常全面的文档。</li>\n</ul>\n<h3 id=\"%E5%9C%A8-ubuntu-%E5%92%8C%E5%85%B6%E4%BB%96-linux-%E5%8F%91%E8%A1%8C%E7%89%88%E4%B8%8A%E5%AE%89%E8%A3%85-glances\" class=\"mume-header\">在 Ubuntu 和其他 Linux 发行版上安装 Glances</h3>\n<p>Glances 在许多 Linux 发行版的官方软件库中都有。这意味着你可以使用你的发行版的软件包管理器来轻松安装它。</p>\n<p>在基于 Debian/Ubuntu 的发行版上，你可以使用以下命令：</p>\n<pre class=\"prettyprint linenums\"><code>sudo apt install glances\n</code></pre>\n<p>你也可以使用 snap 包安装最新的 Glances：</p>\n<pre class=\"prettyprint linenums\"><code>sudo snap install glances\n</code></pre>\n<p>由于 Glances 是基于 Python 的，你也可以使用 PIP 在大多数 Linux 发行版上安装它。先<a href=\"https://itsfoss.com/install-pip-ubuntu/\">安装 PIP</a>，然后用它来安装 Glances：</p>\n<pre class=\"prettyprint linenums\"><code>sudo pip3 install glances\n</code></pre>\n<p>如果没有别的办法，你还可以使用 Glances 开发者提供的自动安装脚本。虽然我们不建议直接在你的系统上随便运行脚本，但这完全取决于你自己：</p>\n<pre class=\"prettyprint linenums\"><code>curl -L https://bit.ly/glances | /bin/bash\n</code></pre>\n<p>你可以从他们的<a href=\"https://github.com/nicolargo/glances/blob/master/README.rst#installation\">文档</a>中查看其他安装 Glances 的方法，甚至你还可以把它作为一个 Docker 容器来安装。</p>\n<h3 id=\"%E4%BD%BF%E7%94%A8-glances-%E7%9B%91%E6%8E%A7%E6%9C%AC%E5%9C%B0%E7%B3%BB%E7%BB%9F%E4%B8%8A%E7%9A%84-linux-%E7%B3%BB%E7%BB%9F%E8%B5%84%E6%BA%90%E7%8B%AC%E7%AB%8B%E6%A8%A1%E5%BC%8F\" class=\"mume-header\">使用 Glances 监控本地系统上的 Linux 系统资源（独立模式）</h3>\n<p>你可以通过在终端上运行这个命令，轻松启动 Glances 来监控你的本地机器：</p>\n<pre class=\"prettyprint linenums\"><code>glances\n</code></pre>\n<p>你可以立即观察到，它将很多不同的信息整合在一个屏幕上。我喜欢它在顶部显示电脑的公共和私人 IP：</p>\n<p><img src=\"https://img.linux.net.cn/data/attachment/album/202008/30/212937p4xg8xuwr9dcoxgo.png\" alt=\"\" /></p>\n<p>Glances 也是交互式的，这意味着你可以在它运行时使用命令与它互动。你可以按&#160;<code>s</code>&#160;将传感器显示在屏幕上；按&#160;<code>k</code>&#160;将 TCP 连接列表显示在屏幕上；按&#160;<code>1</code>&#160;将 CPU 统计扩展到显示单个线程。</p>\n<p>你也可以使用方向键在进程列表中移动，并按不同的指标对表格进行排序。</p>\n<p>你可以通过各种命令行选项来启动 Glances。此外，它还有很多交互式命令。你可以在他们的<a href=\"https://glances.readthedocs.io/en/latest/cmds.html\">丰富的文档</a>中找到完整的列表。</p>\n<p>按&#160;<code>Ctrl+C</code>&#160;键退出 Glances。</p>\n<h3 Glances 监控远程 Linux 系统（客户端-服务器模式）</h3>\n<p>要监控远程计算机，你可以在客户端-服务器模式下使用 Glances。你需要在两个系统上都安装 Glances。</p>\n<p>在远程 Linux 系统上，使用&#160;<code>-s</code>&#160;选项在服务器模式下启动 Glances：</p>\n<pre class=\"prettyprint linenums\"><code>glances -s\n</code></pre>\n<p>在客户端系统中，使用下面的命令在客户端模式下启动 Glances 并连接到服务器：</p>\n<pre class=\"prettyprint linenums\"><code>glances -c server_ip_address\n</code></pre>\n<p>你也可以通过 SSH 进入任何一台电脑，然后启动 Glances，它可以完美地工作。更多关于客户端-服务器模式的信息请看<a href=\"https://glances.readthedocs.io/en/latest/quickstart.html#central-client\">这里</a>。</p>\n<h3 在 Web 浏览器中监控 Linux 系统资源（Web 模式）</h3>\n<p>Glances 也可以在 Web 模式下运行。这意味着你可以使用 Web 浏览器来访问 Glances。与之前的客户端-服务器模式不同，你不需要在客户端系统上安装 Glances。</p>\n<p>要在 Web 模式下启动 Glances，请使用&#160;<code>-w</code>&#160;选项：</p>\n<pre class=\"prettyprint linenums\"><code>glances -w\n</code></pre>\n<p>请注意，即使在 Linux 服务器上，它也可能显示 &#8220;Glances Web User Interface started on&#160;<a href=\"http://0.0.0.0:61208\">http://0.0.0.0:61208</a>&#8221;，而实际上它使用的是服务器的 IP 地址。</p>\n<p>最主要的是它使用的是 61208 端口号，你可以用它来通过网络浏览器访问 Glances。只要在服务器的 IP 地址后面输入端口号，比如&#160;<a href=\"http://123.123.123.123:61208\">http://123.123.123.123:61208</a>。</p>\n<p>你也可以在本地系统中使用&#160;<a href=\"http://0.0.0.0:61208/\">http://0.0.0.0:61208/</a>&#160;或&#160;<a href=\"https://localhost:61208/\">https://localhost:61208/</a>&#160;访问。</p>\n<p><img src=\"https://img.linux.net.cn/data/attachment/album/202008/30/212952u5bph462djb6dkph.png\" alt=\"\" /></p>\n<p>Web 模式也模仿终端的样子。网页版是根据响应式设计原则打造的，即使在手机上也很好看。</p>\n<p>你可能想用密码来保护 Web 模式，这样只有授权的人才能使用它。默认的用户名是&#160;<code>glances</code>。</p>\n<pre class=\"prettyprint linenums\"><code>root@localhost:~# glances -w 最大的优势之一就是开箱即用，它支持将数据导出到各种数据库、服务，并无缝集成到各种数据管道中。</p>\n<p>你可以在监控的同时用这个命令导出到 CSV：</p>\n<pre class=\"prettyprint linenums\"><code>glances --export csv --export-csv-file /tmp/glances.csv\n</code></pre>\n<p><code>/tmp/glances.csv</code>&#160;是文件的位置。数据以时间序列的形式整齐地填入。</p>\n<p><img src=\"https://img.linux.net.cn/data/attachment/album/202008/30/213004w4wy5y0zn500jsif.png\" alt=\"\" /></p>\n<p>你也可以导出到其它大型应用程序，如&#160;<a href=\"https://prometheus.io/\">Prometheus</a>，以启用条件触发器和通知。</p>\n<p>它可以直接插入到消息服务（如 RabbitMQ、MQTT）、流媒体平台（如 Kafka），并将时间序列数据导出到数据库（如 InfluxDB），并使用 Grafana 与其他服务进行整合</h3>\n<p>这是整个栈中我最喜欢的功能。Glances 不仅可以将各种指标汇集在一起，还可以通过 API 将它们暴露出来。</p>\n<p>这个简单而强大的功能使得为任何特定的用例构建自定义应用程序、服务和中间件应用程序变得非常容易。</p>\n<p>当你在 Web 模式下启动 Glances 时，REST API 服务器会自动启动。要在 API 服务器模式下启动它，你可以使用以下命令：</p>\n<pre class=\"prettyprint linenums\"><code>glances -w --disable-webui\n</code></pre>\n<p><a class=\"prettyprint linenums\"><code>$ stat ckacct\n  File: ckacct\n  Size: 200             Blocks: 8          IO Block: 4096   regular file\nDevice: 801h/2049d      Inode: 829041      Links: class=\"mume-header\">按组别列出文件</h3>\n<p>要按关联的组别对文件进行排序，你可以将一个长列表的输出传给&#160;<code>sort</code>&#160;命令，并告诉它在第 4 列进行排序。</p>\n<pre class=\"prettyprint linenums\"><code>$ ls -l | sort -k4\n</code></pre>\n<h3 id=\"%E6%8C%89%E8%AE%BF%E9%97%AE%E6%97%A5%E6%9C%9F%E5%88%97%E5%87%BA%E6%96%87%E4%BB%B6\" class=\"mume-header\">按访问日期列出文件</h3>\n<p>要按访问日期（最近访问的日期在前）列出文件，使用&#160;<code>-ltu</code>&#160;选项。<code>u</code>&#160;强制&#8220;按访问日期&#8221;排列顺序。</p>\n<pre class=\"prettyprint linenums\"><code>$ ls id=\"%E5%8D%95%E8%A1%8C%E5%88%97%E5%87%BA%E5%A4%9A%E4%B8%AA%E6%96%87%E4%BB%B6\" class=\"mume-header\">单行列出多个文件</h3>\n<p>有时，精简的文件列表更适合手头的任务。<code>ls</code>&#160;命令甚至有这方面的选项。为了在尽可能少的行上列出文件，你可以使用&#160;<code>--format=comma</code>&#160;来用逗号分隔文件名，就像这个命令一样：</p>\n<pre class=\"prettyprint linenums\"><code>$ ls --format=comma\n1, 10, 11, 12, 124, 13, 14, 15, 16pgs-landscape.pdf, 16pgs.pdf, 17, 18, 19,\n192.168.0.4, 2, 20, 2018-12-23_OoS_2.pdf, 2018-12-23_OoS.pdf, 20190512_OoS.pdf,\n'2019_HOHO_application working.pdf' &#8230;\n</code></pre>\n<p>喜欢用空格？使用&#160;<code>--format=across</code>&#160;代替。</p>\n<pre class=\"prettyprint linenums\"><code>$ ls --format=across z*\nz              zip            zipfiles      zipfiles1.bat   zipfiles2.bat\nzipfiles3.bat  zipfiles4.bat  zipfiles.bat  zoom_amd64.deb  zoomap.pdf\nzoom-mtg\n</code></pre>\n<h3 id=\"%E5%A2%9E%E5%8A%A0%E6%90%9C%E7%B4%A2%E7%9A%84%E6%B7%B1%E5%BA%A6\" class=\"mume-header\">增加搜索的深度</h3>\n<p>虽然&#160;<code>ls</code>&#160;一般只列出单个目录中的文件，但你可以选择使用&#160;<code>-R</code>&#160;选项（<ruby>递归<rt>Recursively</rt></ruby>）地列出文件，深入到整个目录的深处。</p>\n<pre class=\"prettyprint linenums\"><code>$ ls -R zzzzz | grep -v \"^$\"\nzzzzz:\nzzzz\nzzzzz/zzzz:\nzzz\nzzzzz/zzzz/zzz:\nzz\nzzzzz/zzzz/zzz/zz:\nz\nzzzzz/zzzz/zzz/zz/z:\nsleeping\n</code></pre>\n<p>另外，你也可以使用&#160;<code>find</code>&#160;命令，对深度进行限制或不限制。在这个命令中，我们指示&#160;<code>find</code>&#160;命令只在三个层次的目录中查找：</p>\n<pre class=\"prettyprint linenums\"><code>$ find zzzzz -maxdepth 3\nzzzzz\nzzzzz/zzzz\nzzzzz/zzzz/zzz\nzzzzz/zzzz/zzz/zz\n</code></pre>\n<h3 id=\"%E9%80%89%E6%8B%A9-ls-%E8%BF%98%E6%98%AF-find\" class=\"mume-header\">选择 ls 还是 find</h3>\n<p>当你需要列出符合具体要求的文件时，<code>find</code>&#160;命令可能是比&#160;<code>ls</code>&#160;更好的工具。</p>\n<p>与&#160;<code>ls</code>&#160;不同的是，<code>find</code>&#160;命令会尽可能地深入查找，除非你限制它。它还有许多其他选项和一个&#160;<code>-exec</code>&#160;子命令，允许在找到你要找的文件后采取一些特定的行动。</p>\n<h3
