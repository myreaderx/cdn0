Check out the video above for complete demonstrations of these features—here’s a quick summary:</p>\n<h3>Padding</h3>\n<p>Imagine you create a text object for a button. You add the text to the workspace, then <strong>right-click &gt; Auto Layout</strong>. Doing so immediately creates a frame around the text object (see the layers panel), applies a default 10px of padding <em>between</em> the items (we’ll get to this in a moment) and a default 10px of padding around the object (see the properties panel):</p>\n<figure class=\"post_image\"><img src=\"https://cms-assets.tutsplus.com/uploads/users/30/posts/36260/image-upload/padding.jpg\" alt=\"default padding on auto layout frame\"></figure>\n<p>These values can be changed by entering different numbers. In the old version of Figma Auto Layout you could apply padding by specifying horizontal and vertical values, but in this new Auto Layout you can enter values for top, bottom, left, and right, all within the <strong>Alignment and Padding</strong> panel:</p>\n<figure class=\"post_image\"><img src=\"https://cms-assets.tutsplus.com/uploads/users/30/posts/36260/image-upload/alighment-padding.jpg\" alt=\"alignment and padding\"></figure>\n<h3>Resizing</h3>\n<p>We also have some exciting new options for resizing. For example, we can apply Auto Layout to a button (just like above) then in the <strong>Resizing</strong> properties we can apply <strong>Hug Contents</strong> both vertically, and horizontally. This makes the parent frame “hug” the button by resizing itself depending on how big the button is</p>\n<p>Alternatively we can set this resize value to <strong>Fixed Width</strong> and <strong>Fixed Height</strong>. As you might expect, this allows you to enter a fixed value for the width and height of the button.</p>\n<p>The last possible value here is <strong>Fill Container</strong>, which will make the button as wide or as tall as the available space in the parent container allows. </p>\n<p>These three approaches to the sizing of objects, especially as you can use any combination you like on one object, allow total control when dealing with whole page and component layouts.</p>\n<h3>Alignment</h3>\n<p>The new Figma Auto Layout offers us some new alignment control too. In the old version of Auto Layout you would perform any alignment at the “child level”, ie. on the objects themselves. Now, you apply alignment properties to the parent container.</p>\n<p>You can see the how alignment is impacting the child objects in the Alignment and Padding panel we looked at earlier:</p>\n<figure class=\"post_image\"><img src=\"https://cms-assets.tutsplus.com/uploads/users/30/posts/36260/image-upload/align.jpg\" alt=\"alignment of objects\"></figure>\n<p>In the image above you can see that objects within this frame are aligned to the top left. You can click anywhere in this space to change the alignment visually.</p>\n<p>Personally I quite liked the older version where I could align objects individually, but I can see how this approach makes sense.
