As always, many bug fixes are included, too.</p>\n\n\n\n<h2>Improving the Block toolbar</h2>\n\n\n\n<p>In order to provide a consistent UI across blocks, blocks have seen their toolbars rearranged following a <a href=\"https://github.com/WordPress/gutenberg/issues/25983\">standard grouping layout</a>. While the last release saw the Image block have its toolbar standardized, this time a whole set of blocks have been normalized as well, including blocks such as Button, Buttons, List, Heading, Paragraph, Quote, Audio, File, Media and Text, Video, Site Logo, Cover, and Post Featured Image blocks.</p>\n\n\n\n<figure class=\"wp-block-image\"><img src=\"https://cldup.com/77LupMjz-2-3000x3000.jpeg\" alt=\"\"></figure><h2>Introducing layout configuration in theme.json</h2>\n\n\n\n<p>Themes have always had the responsibility of defining alignments in the frontend through CSS rules. They also define different widths for both the editor and frontend. The content width, whether they support wide alignments or not, and the CSS to support all these combinations can be very hard to write properly.</p>\n\n\n\n<p>With Full Site Editing, the templates are written in blocks and edited in the site editor. This means that the editor has to match the front end as much as possible while allowing theme authors to tweak these alignments and widths in the editor. If this doesn’t happen, it causes confusion and frustration as seen <a href=\"https://make.wordpress.org/core/2021/03/16/high-level-feedback-from-the-fse-program-march/\">in the feedback from the FSE Outreach Program</a>. </p>\n\n\n\n<p>To address these requirements and to simplify the way themes define and style alignments, Gutenberg 10.3 introduces the concept of <strong>layout</strong> and layout configuration, a feature automatically enabled when you rely on a <code>theme.json</code> file (<code>experimental-theme.json</code> file at the moment).</p>\n\n\n\n<p>Theme authors should be able to make use of this feature by adding a <code>layout</code> config to their <code>theme.json</code> file and indicating in their block templates which containers inherit that config. The <code>layout</code> config is also automatically applied to the post editor.</p>\n\n\n\n<p>If you’re using the experimental <code>theme.json</code> file already, you should be able to follow these two pull requests that update the<a href=\"https://github.com/WordPress/theme-experiments/pull/233\"> empty theme</a> and the<a href=\"https://github.com/WordPress/theme-experiments/pull/236\"> twenty twenty-one blocks theme</a> to use this feature.</p>\n\n\n\n<h2>Changes in theme.json block supports and supported styles </h2>\n\n\n\n<p>Block supports are no longer used to decide whether the styles in the theme.json file for a block should be used or not. However, block supports are still used to show UI controls in both the block &#38; global styles sidebar, but themes can now use any theme.json style property that exists in any block:</p>\n\n\n\n<pre class=\"wp-block-code\"><code class=\"\">\"core/paragraph\": {\n  \"typography\": {\n    \"fontFamily\": \"var(--wp--preset--font-family--cambria-georgia)\"\n  }\n}</code></pre>\n\n\n\n<h2>Improvements to Navigation Block and Navigation Editor</h2>\n\n\n\n<p>Gutenberg 10.3 brings a number of improvements to the Navigation block and Navigation editor experience, including the <a href=\"https://github.com/WordPress/gutenberg/pull/29936\">list view in the Navigation editor</a>, <a href=\"https://github.com/WordPress/gutenberg/pull/28833\">in-between inserter</a>, and a <a href=\"https://github.com/WordPress/gutenberg/pull/29713\">better flow to build nested menus</a>.</p>\n\n\n\n<figure class=\"wp-block-image\"><img src=\"https://user-images.githubusercontent.com/1204802/110622076-47d53080-819b-11eb-8893-54ffe04f1b21.gif\" alt=\"after, editor\"></figure><h2>Inserter enhancements</h2>\n\n\n\n<p>This release also brings updates to inserting blocks, including improvements to <a href=\"https://github.com/WordPress/gutenberg/pull/26938\">keyboard navigation and accessibility</a> for the Block Inserter, and a <a href=\"https://github.com/WordPress/gutenberg/pull/30020\">new “theme” category for template parts</a> and their variations. Moreover, the slash inserter now allows <a href=\"https://github.com/WordPress/gutenberg/pull/29939\">typing multiple words when searching for blocks</a>.
