Manjaro</a>&#160;时，它给出了从开源（自由）或非开源（非自由）图形驱动安装开始的选项。当你选择&#8220;非自由&#8221;时，它会自动检测你的显卡，并为其安装最合适的驱动程序，因此 GPU 可以开箱即用。</p>\n<p>由于有了上一节中看到的硬件检测工具，甚至在安装 Manjaro 时，安装显卡驱动会更加容易。</p>\n<p>如果你有一个带有 Nvidia Optimus 卡（混合 GPU）的系统，它与 Manjaro 配合良好。你会有很多方式来让它工作。</p>\n<p>在 Arch Linux 中，你必须为你的机器安装（如果你能找到）合适的驱动程序。</p>\n<h4 id=\"%E8%AE%BF%E9%97%AE-arch-%E7%94%A8%E6%88%B7%E8%BD%AF%E4%BB%B6%E5%BA%93aur\" class=\"mume-header\">访问 Arch 用户软件库（AUR）</h4>\n<p><a href=\"https://linux.cn/article-12107-1.html\">Arch 用户软件库</a>（AUR）是一个面向基于 Arch 的 Linux 发行版用户的社区驱动的软件库。AUR 的创建是为了组织和分享来自社区的新软件包，并帮助加快流行软件包被纳入<a href=\"https://wiki.archlinux.org/index.php/Community_repository\">社区软件库</a>。</p>\n<p>大量进入官方软件库的新软件包都是从 AUR 开始的。在 AUR 中，用户能够贡献自己的软件包构建（<code>PKGBUILD</code>&#160;和相关文件）。</p>\n<p>你可以在 Arch 和 Manjaro 中使用 AUR。</p>\n<h4 id=\"%E6%A1%8C%E9%9D%A2%E7%8E%AF%E5%A2%83\" class=\"mume-header\">桌面环境</h4>\n<p>好吧！你可以在任何 Linux 发行版上使用几乎所有的桌面环境。Arch 和 Manjaro 也不例外。</p>\n<p>然而，一个专门的桌面风格或版本可以让用户更容易地在桌面环境里获得顺畅的体验。</p>\n<p>默认的 Arch ISO 并不包含任何桌面环境。例如，你想<a href=\"https://linux.cn/article-12258-1.html\">在 Arch Linux 上安装 KDE</a>，你必须在<a href=\"https://itsfoss.com/install-arch-linux/\">安装 Arch Linux</a>&#160;时或在之后下载安装它。</p>\n<p>而 Manjaro 则为 Xfce、KDE 和 GNOME 等桌面环境提供了不同的 ISO。Manjaro 社区还维护着 MATE、Cinnamon、LXDE、LXQt、OpenBox 等桌面环境的 ISO。</p>\n<h4 id=\"%E5%AE%89%E8%A3%85%E7%A8%8B%E5%BA%8F\" class=\"mume-header\">安装程序</h4>\n<p><img src=\"https://img.linux.net.cn/data/attachment/album/202008/18/110750y89zhvhzlouhhjkn.jpg\" /></p>\n<p>Manjaro 是基于 Arch Linux 的，它是兼容 Arch 的，但<strong>它不是 Arch</strong>。它甚至不是只有一个图形安装程序的预配置版本的 Arch。Arch 并不具备通常的舒适的开箱即用，这也是为什么大多数人喜欢更简单的东西。Manjaro 为你提供了简单的入口，但支持你成为经验丰富的用户或资深用户。</p>\n<h4 id=\"%E6%96%87%E6%A1%A3%E5%92%8C%E6%94%AF%E6%8C%81\" class=\"mume-header\">文档和支持</h4>\n<p>Arch 和 Manjaro 都有自己的维基页面和支持论坛来帮助各自的用户。</p>\n<p>虽然 Manjaro 有一个不错的<a href=\"https://wiki.manjaro.org/index.php?title=Main_Page\">维基</a>文档，但&#160;<a href=\"https://wiki.archlinux.org/\">Arch 维基</a>则不可同日而语。你可以在 Arch 维基中找到关于 Arch Linux 各方面的详细信息。</p>\n<h4 id=\"%E7%9B%AE%E6%A0%87%E5%8F%97%E4%BC%97\" class=\"mume-header\">目标受众</h4>\n<p>关键的区别在于&#160;<a href=\"https://linux.cn/article-12445-1.html\">Arch 针对的是抱着自己动手的态度的用户</a>，他们愿意阅读文档，自己解决问题。</p>\n<p>而 Manjaro 则是针对那些没有那么多经验或者不想花时间组装操作系统的 Linux 用户。</p>\n<h3 id=\"%E7%BB%93%E8%AE%BA\" class=\"mume-header\">结论</h3>\n<p>有些人经常说 Manjaro 是给那些不会安装 Arch 的人用的。但我认为这是不对的。不是每个人都想从头配置 Arch，或者没有太多时间。</p>\n<p>Manjaro 绝对是一只野兽，但与 Arch 截然不同。<strong>快速、强大，并总是保持更新</strong>，Manjaro 提供了 Arch 操作系统的所有优点，但特别强调<strong>稳定性、用户友好性和可访问性</strong>，既适合新手，也适合有经验的用户。</p>\n<p>Manjaro 并不像 Arch Linux 那样极简主义。在 Arch 中，你从一个空白的画布开始，手动调整每个设置。当默认的 Arch 安装完成后，你在命令行就有了一个正在运行的 Linux 实例。想要一个<a href=\"https://itsfoss.com/best-linux-desktop-environments/\">图形化桌面环境</a>？那就自己来吧 &#8212;&#8212; 有很多选择。选择一个，安装，然后配置它。你可以从中学到很多东西，特别是如果你是 Linux 新手的话。你会对系统是如何组合在一起的，以及为什么要以这样的方式安装东西有很好的理解。</p>\n<p>我希望你现在对 Arch 和 Manjaro 有了更好的理解。现在，你明白了它们是相似而不同的了吧。</p>\n<p>我已经发表了我的看法。不要犹豫，请在评论区分享你的观点。在 Arch 和 Manjaro 之间，你更喜欢哪一个，为什么。</p>\n<p>Abhishek Prakash 也对此文补充了内容。</p>\n<hr />\n<p>via:&#160;<a
