style=\"margin-left:0px;\">看板类服务其实在效率工具中并不少见，最为出名的当属 Trello —— 设置各种不同作用的列表来对不同的事项进行分类，而通过在列表中添加各种形式的卡片来设置相关的工作事项，并将其分配给各个成员来对工作进行分配和安排。对于一些轻沟通的团队而言，看板应用结合丰富的工具组合，可以有效降低沟通成本，实现效率工具利用的最大化。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/99da510d065e35de01ea68043b80da05.png\"><figcaption>选择不同类型的看板模版 来源：Microsoft</figcaption></figure><p>因此微软最终在 Outlook 线上版本中悄悄加入了这个名为 Project Moca 看板服务，虽然形态上很像我们看到的看板服务，<strong>但在我看来 Project Moca 可能更接近看板应用和白板应用的结合体</strong>。</p><p>传统的看板应用中更像是文件柜，文件需要时各自认领、而白板应用过于自由发散，在一定程度上并不适合团队协作分工。Project Moca 则更接近所谓「会议桌应用」——每一个人都可以自由的将相关的资料、工具等放在 Project Moca 这个「大桌子」上，然后根据需要自由领取。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/84a6e7e7e03bdba1a965231520c16861.PNG\"><figcaption>摊开在画布上的各种卡片 来源：Microsoft</figcaption></figure><p>这就是 Project Moca 的业务逻辑，虽然看板整体样式和传统看板类似，但创建「业务卡片」时的思路却又不太一样：首先所有的「列表」均叫做「存储桶」，可以建立的卡片类型则会出现在左侧的工具箱中，包括待办、文档、地图、网页解链接甚至天气等等——想象下真实的办公室会议场景，这种看板模式是不是很像开会时把所有需要用到的资料都摊放在办公桌上给同仁审阅？</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/3e73832d5c7cd294220628f6e893bc6c.png\"><figcaption>在画布中创建卡片并进行设置</figcaption></figure><p><strong>因此 Project Moca 在一定程度上更像是有看板应用之形的数字会议桌</strong>，比如你可以建立起包括各种文件附件的卡片，在进行小组讨论时可以将其摊放在画布之上供成员查看，后续会议结束再将其归纳到某个「存储桶」中；也可以设置有着天气、任务、备注的卡片，并将其组合放在某个存储桶中，事情处理完成之后再将整个存储桶从画布中删除。这种高自由度也得益于微软提供了各种独立的服务，而 Project Moca 只是有机地将其整合在了一起。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/3c7bac44ffa24968785f936b82718305.png\"><figcaption>以天为单位设定行动规划的「资源桶」来源：Microsoft</figcaption></figure><p>也正因为存储桶中的各类卡片其实都基于微软现有的服务，最终在 Project Moca 中建立的卡片状态都会被同步到原先的各个效率工具服务中。即便之后不再打开 Project Moca 的看板，我们依旧可以在其他独立的效率工具中去完成看板中安排的各种事项。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/e480222de5a2b479478a3a229b2d1465.png\"><figcaption>Project Moca 设置的卡片会同步到各个独立应用中 图：Microsoft</figcaption></figure><p style=\"margin-left:0px;\">Project Moca 或许正是微软数字化协作的未来形态。它像一个物理具象化意味更强、既可以拖拽也可以无限延展的「画布」，协作者将各种任务卡片堆砌在画布上，参加协作办公的人员再根据实际工作安排和需要，将其规整、同步到自己的「存储桶」和效率工具当中。</p><figure class=\"image ss-img-wrapper\"><img src=\"https://cdn.sspai.com//2020/09/05/28afc2b65190705d6d12fdca3cd11c52.png\"></figure><p>这种协作形态也在一种程度上勾勒出微软眼中的「生产力场景」：通过 Office 文档来完成文档以及办公内容的创建和制作，各种效率协作工具则作为会议沟通的工具，而 Project Moca 则是最佳的会议沟通的平台，根据会议内容再进行工作内容的再分配和协作，最终会议的执行和成果交给各个独立的效率工具来继续完成反馈和溯源，或许在某些场景上远比 Microsoft Team 中要简单直接得多。</p><p>最后，其实从名字也不难看出，Project Moca 目前依旧是微软正在进行测试的新工具，因此其并没有独立应用且目前只是作为 Outlook Web App 的一个子功能，但其实经过简单的试用之后已经可以看出微软对于其各个效率工具的有了新的看法和组织形式，相比其他看板需要通过扩展来实现跨应用来协作，Project Moca 背后微软丰富的「效率库」让其在协作过程中更为顺畅。</p><p>类似的「模块化」整合与协同思路此前我们在微软的 Fluid Framework 中其实已经看到过了，对于依赖微软账号体系的办公族来说，运用了类似思路的 Project
