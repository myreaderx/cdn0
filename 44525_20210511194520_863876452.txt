Why not Excel?</h3>\n\n\n\n<p>我需要先解释下为什么没有继续使用 Excel 来可视化。</p>\n\n\n\n<p>其实，微软三件套中，Excel 的功能最为强大，定制能力最强，Excel 自带 VBA 编程环境，而且各种模版，各种函数，用好了就是一件大杀器，但遗憾的是，Excel Mac 版本的使用体验并不理想，暂且不说打开数据表格的速度要比 Windows 慢太多，更遗憾的是，Mac Excel 还是一个阉割版，比如 Mac 上就没有 Power Query 功能。</p>\n\n\n\n<p>所以有必要再选一个 Mac 平台上的可视化工具。</p>\n\n\n\n<h3>2. Tableau：不止于可视化，更是BI工具</h3>\n\n\n\n<p>市面上可视化工具很多，国内也有很多网页端工具，方便，快捷。但我还是选择了 Tableau，首先它很好上手，其次它除了最基础的出图功能外，还提供了很多分析功能，也因此更接近一件半自动 BI（商业智能）分析工具。</p>\n\n\n\n<blockquote class=\"wp-block-quote\"><p>Tableau 可以轻松创建和分发交互式数据仪表板，通过简单而有效的视觉效果提供对动态，变化趋势和数据密度分布的深入描述，Tableau 很容易理解为工作工具，其学习曲线非常温和，因为它努力为任何类型的用户提供其所有权力，甚至是那些以前从未接触过可视化工作流技术细节的用户。</p></blockquote>\n\n\n\n<p>对于工具软件来说，我认为大体分为两类：</p>\n\n\n\n<ul><li>将业余「专业化」的工具</li><li>将专业「业余化」的工具</li></ul>\n\n\n\n<p>前者如抖音、vue 这些视频工具，人们通过简单的操作也可以得到不错的效果。（业余专业化）</p>\n\n\n\n<p>而 Tableau 则是面对那些不怎么具备专业技术的普通人员，让他们也可以通过简单的拖拽生成图表，它降低了数据分析的门槛。（专业业余化）</p>\n\n\n\n<p>Tableau 是一个收费软件，不过我们可以下载它的<a href=\"https://public.tableau.com/zh-cn/s/\">免费 public 版本</a>。</p>\n\n\n\n<p>下载好了之后，我们打开软件，我这里使用 desktop 举例，二者区别不大。</p>\n\n\n\n<h3>3. Workflow</h3>\n\n\n\n<p>本文的初衷，是希望<strong>通过实践建立起一个简单的数据可视化 Workflow。</strong>因为在我看来，每个人都应该具有选择工具的权利与能力，来突破隔行如隔山的专业性「代沟」，增强自己的竞争力。</p>\n\n\n\n<p>就算在少数派这样重实操，较极客的平台，对大部分人来说，编程还是有门槛，</p>\n\n\n\n<p>任何一件事都有门槛，但我们能做的，就是不断把这个门槛降低，让更多的人感受「data is beautiful」。</p>\n\n\n\n<h3>2. 工具潜力未被完全挖掘</h3>\n\n\n\n<p>篇幅有限，本文对工具的介绍并不全面，也就是说它们还可以达到更高的水平。</p>\n\n\n\n<p>比如我们想要对 Matrix 文章的调性做一个数据分析，其实 Web Scraper 还可以自动抓取跳转链接后的数据，也就可以通过 Web Scraper 跳转文章标题链接，拿到文章页面中「标签」这一维度的数据，从而分析出作者都爱写哪些类型文章，读者又都更偏爱哪些类型文章等。</p>\n\n\n\n<p>再比如我们可以让 Web Scraper 跳转作者页面，拿到作者的总写作字数，这个时候就可以配合 Tableau 的参数化<strong>计算字段</strong>，找到一些新角度，比如计算<strong>点赞数/写作字数</strong>，就可以看出作者驾驭文字的能力。</p>\n\n\n\n<p>这样的角度还有很多，等着你去挖掘。</p>\n\n\n\n<h3>3. 工具的局限性</h3>\n\n\n\n<p>再好用的工具也有自己的局限。</p>\n\n\n\n<p>相比于 Python、D3、R、Echart 这些「正统」数据处理语言来说，Web Scraper 首先<strong>缺少实时性</strong>，也就是说它不能每时每刻监控网页上的数据变动，每次爬取数据都要手动开始，也不能提前定时，<strong>自动化水平不高</strong>，而且数据采集要在 Chrome 中模拟翻页，点击，<strong>耗时较长</strong>。而且 Web Scraper 和 Tableau 都有一定的<strong>性能瓶颈</strong>，当需要处理的是百万，千万级别的数据时，软件的响应速度远远不如编程语言。</p>\n\n\n\n<p>所以说，工具的出现并不意味编程过时了，只是这些工具也能让普通人上手一些<strong>轻场景</strong>的数据分析，比如做新媒体，需要分析竞品网站，这个时候工具的优势就体现出来了，从数据采集到分析，都不太需要过多的编程知识。</p>\n\n\n\n<p>而对于那些数据量级巨大，网页爬取不便，且对可视化出图样式要求较高的<strong>重场景</strong>数据分析来说，编程工具会更合适。</p>\n\n\n\n<p>最后，简单谈下我对数据可视化的理解。</p>\n\n\n\n<h2>六、微谈数据可视化</h2>\n\n\n\n<h3>1. 数据可视化的本质</h3>\n\n\n\n<p>数据可视化有着统计学渊源，但并不是只有严格意义上的传统图表才称得上可视化。不管是我们演示用的图例，还是基于气象模型的 R 语言的专业数据分析，或是好奇心日报的「好奇心小数据」这样的数据新闻，其实都算是数据可视化。</p>\n\n\n\n<p>那可视化的本质是什么？</p>\n\n\n\n<p>在知名可视化作者 <a href=\"https://book.douban.com/search/Nathan%20Yau\">Nathan Yau</a> 看来：</p>\n\n\n\n<blockquote class=\"wp-block-quote\"><p>我们手头的大堆数据反映了真实的世界，然后我们对它们进行分析，得到的不只是视数据的关联性，我们还能了解到身边正在发生什么。这些故事反过来可以帮助我们解决真实世界中存在的问题，例如降低犯罪率、提高卫生意识、改善高速上的交通状况，或者只是增长我们的见识。</p></blockquote>\n\n\n\n<p>数据可视化的本质是<strong>寻求真相</strong>，透过数据——世界的真实反应，来找寻各种事件间的相关性与事情发展的趋势，从而找到一些问题，然后我们才会想方设法解决这些问题，让这个世界变得更好。</p>\n\n\n\n<p>寻找问题，解决问题，把你的想法通过图表顺畅的传达给读者，我认为这就是数据可视化的第一要义。</p>\n\n\n\n<p>从本文前几章的工具教程可以看出，工具的上手难度并不高，但是如何着手做好数据可视化却很难，换句话说，可视化的「术」很简单，「道」却很难。</p>\n\n\n\n<h3>2. 「道」即是问题</h3>\n\n\n\n<blockquote class=\"wp-block-quote\"><p>图片最伟大的价值在于它迫使我们注意到从未预见到的事物。——约翰·图基</p></blockquote>\n\n\n\n<p>当我着手对 Matrix 的文章可视化时，我需要通过不断<strong>设问</strong>来推进可视化：</p>\n\n\n\n<ol><li>我可以拿到什么数据？</li><li>我想了解数据的哪些点？</li><li>数据背后的问题是什么？我对哪些问题有疑问？</li><li>选择哪些合适的图表？</li></ol>\n\n\n\n<figure class=\"wp-block-image\"><a href=\"https://podcast.webview.tech/podblog/2019-04-26-085032.png\" data-rel=\"lightbox-image-65\" data-rl_title=\"\" data-rl_caption=\"\" title=\"\"><img src=\"https://podcast.webview.tech/podblog/2019-04-26-085032.png\" alt=\"\"/></a><figcaption>来源：《数据之美》</figcaption></figure>\n\n\n\n<p><strong>我的回答</strong>：</p>\n\n\n\n<ol><li>文章标题、作者、点赞、评论、发布日期</li><li>最好的是谁？勤勉（发文数）、受欢迎（点赞）等各种排名，时间的变化呢？</li><li>先从整体看数据，然后放大到具体的分类和独立的数据点，哪里增加、减少、离群值，数据中的差异与随机性相比是怎样的？</li><li>柱形图（排名）、折线图（增长率）、堆叠图/饼图（首页比例）等</li></ol>\n\n\n\n<p>当然，在可视化的过程中，你最好围绕一个核心问题出发，有时核心问题来源于你的核心需求，比如对于 Matrix，<strong>我的核心问题是：如何写一篇受大家欢迎的文章？</strong></p>\n\n\n\n<p>这个问题带到数据解读中，又可以转换成：<strong>什么样的文章最受大家欢迎？</strong></p>\n\n\n\n<p>我们再回顾下单篇最受欢迎文章：</p>\n\n\n\n<figure class=\"wp-block-image\"><a href=\"https://podcast.webview.tech/podblog/2019-04-26-085034.png\" data-rel=\"lightbox-image-66\" data-rl_title=\"\" data-rl_caption=\"\" title=\"\"><img src=\"https://podcast.webview.tech/podblog/2019-04-26-085034.png\" alt=\"\"/></a><figcaption>指南、教程类文章榜上有名</figcaption></figure>\n\n\n\n<p>我们可以看到，最受大家欢迎的文章<strong>高度集中在「指南」、「教程」</strong>中，一方面这说明少数派的读者都有很强的学习欲望，从而对教程类文章更感兴趣，另一方面，大家送出的「赞」在这些文章中更像是「收藏」，即某种标记，从而可以在需要的时候找出来再看一遍。</p>\n\n\n\n<p>这里说句题外话，很多互联网产品，都会将「喜欢」、「赞」与「收藏」联系在一起，会更符合一些用户的使用习惯。（我个人就是把赞当收藏用）</p>\n\n\n\n<p>比如第三方微博客户端，墨客：</p>\n\n\n\n<div class=\"wp-block-image\"><figure class=\"aligncenter is-resized\"><a href=\"https://podcast.webview.tech/podblog/2019-04-26-085036.jpg\" data-rel=\"lightbox-image-67\" data-rl_title=\"\" 你不一定需要炫酷图表</h3>\n\n\n\n<p>有时我们会见到一些炫酷的图表，比如桑葚图：</p>\n\n\n\n<div class=\"wp-block-image\"><figure class=\"aligncenter\"><a href=\"https://podcast.webview.tech/podblog/2019-04-26-085037.png\"
