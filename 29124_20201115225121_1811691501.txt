When you select a code fragment you want to extract and press ⌥⌘M or select <em>Refactor / Extract/Introduce / Method&#8230;</em>, the IDE will refactor it immediately without displaying a dialog window. </p>\n<p>After performing this refactoring, you can click the gear icon next to the method name and select additional settings in the drop-down list. The IDE will then open the downloaded pod in the editor. </p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_PodLogs2.gif\" alt=\"\" /></p>\n<p>By default, your pods will be saved to <em>Scratches and Consoles | Kubernetes Files | &#8220;context&#8221; | &#8220;namespace&#8221; | pods</em>. </p>\n<p>There are several settings you can edit to customize the download process.  </p>\n<p>You can specify the download path for pod logs by going to <em>Preferences / Settings | Build, Execution, Deployment | Kubernetes</em> and changing it in the <em>Pod Logs </em>section. Notice that <em>&#8220;context&#8221; | &#8220;namespace&#8221; | pods</em> will be added to the specified path automatically.</p>\n<p>You can also choose to specify the path every time a download commences. In this case, the logs will be saved to the exact location that you specify. </p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_SaveLogs.png\" alt=\"\" /></p>\n<p>Another option in this section allows you to version your logs. If you tick the <em>Append timestamp to pod log name when download </em>checkbox<em>, </em>then each downloaded pod log will be saved separately with a timestamp in the file name. If this checkbox is clear, the IDE will overwrite the log with each new download. </p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_timestamp.png\" alt=\"\" /></p>\n<h3 id=\"resources_deletion\">Resources deletion </h3>\n<p>It is now possible to quickly delete any of your Kubernetes resources, either by clicking the garbage bin icon in the left pane of the <em>Services</em> tool window or by selecting <em>Delete Resource</em> from the context menu. </p>\n<h3 id=\"open_console_and_run_shell\">Open Console and Run Shell </h3>\n<p>You can connect containers that include consoles via <em>Open Console</em>, which you can invoke from the context menu or by clicking the <em>Open Console </em>button in the <em>Services</em> tool window.</p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_Console.png\" alt=\"\" /></p>\n<p>If you want to launch Shell for a pod container, select <em>Run Shell </em>from the context menu or click the Run Shell button in the left pane of the Services tool window. The launched shell will then open in a new tab.</p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_RunShell.gif\" alt=\"\" /></p>\n<p>You can click the tools icon in the left pane of the <em>Services</em> tool window for quick access to Kubernetes settings. In the <em>Pod Shell </em>section, define which command to use for running a shell inside a container (by default, it is /bin/<em>bash</em>) and choose whether you want to apply this command to all projects or just to the current one. </p>\n<h3 id=\"load_crds_from_kubernetes\">Load CRDs from Kubernetes </h3>\n<p>Your CRD schemas provide a set of rules for creating Kubernetes resources. Currently, it is possible to load CRDs from files or from URLs. This is great, but CRDs can vary by cluster or context. For example, you can have three contexts with three different clusters. These clusters do not have the same CRDs, and they may have different versions as well. </p>\n<p>In v2020.3, we’ve implemented a solution for such cases. In <em>Preferences / Settings | Languages &#38; Frameworks | Kubernetes, </em>you can select the <em>Use API schema from an active cluster</em> checkbox to automatically load CRD schemas from an active cluster, if there are any.</p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/Kubernetes_ApiSchema.png\" alt=\"\" /></p>\n<h2 id=\"kotlin\">Kotlin </h2>\n<h3 id=\"structural_search_and_replace\">Structural search and replace </h3>\n<p>In IntelliJ IDEA 2020.3 EAP7, we’ve introduced support for <a href=\"https://www.jetbrains.com/help/idea/structural-search-and-replace.html\">s</a><a href=\"https://www.jetbrains.com/help/idea/structural-search-and-replace.html\">tructural search and replace (SSR)</a><em> </em>actions for Kotlin. To enable this feature, click the gear icon in the <em>Project</em> view and select both <em>Enable Preview Tab </em>and <em>Open Files with Single Click</em>. If you start editing a file opened in this way, it will cease to be a preview and will become an ordinary file. </p>\n<p><img src=\"https://blog.jetbrains.com/wp-content/uploads/2020/11/preview_tabs.gif\" alt=\"\" /></p>\n<h3 id=\"setting_intellij_idea_as_the_default_application_for_opening_files\">Setting IntelliJ IDEA as the default application for opening files </h3>\n<p>Starting with v2020.3, you can make IntelliJ IDEA a default application for opening files.
