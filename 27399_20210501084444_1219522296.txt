Also, it may help to uninstall JupyterLab 2 if you already have it installed.)</p><p>Here is an overview of the major user-facing changes in JupyterLab 3.0.</p><h3>Debugger</h3><p><a href=\"https://blog.jupyter.org/a-visual-debugger-for-jupyter-914e61716559\">Announced earlier this year</a>, the visual debugger is now shipped by default with JupyterLab.</p><p>To use the visual debugger, you will need a kernel that supports debugging. An example of such kernel is <a href=\"https://xeus-python.readthedocs.io\">xeus-python</a>, the first Jupyter Kernel for Python with support for debugging.</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*RoJnb1XpB6g5geG15WgrMg.gif\" /><figcaption>Stepping into Python code in JupyterLab with the visual debugger</figcaption></figure><p>Check out the <a href=\"https://jupyterlab.readthedocs.io/en/stable/user/debugger.html\">user documentation</a> for more information, including a notebook based tutorial to interactively learn how to use the debugger.</p><h3>Table of Contents</h3><p>The table of contents extension now ships with JupyterLab. This makes it easy to see and navigate the structure of a document.</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*bSH9IcyWHLaNML2q9lRThg.gif\" /><figcaption>Using the Table of Content in JupyterLab 3.0</figcaption></figure><h3>Support for multiple display languages</h3><p>JupyterLab now provides the ability to set the display language of the user interface.</p><p>Users will need to install the language pack as a separate Python package. Language packs are grouped in the <a href=\"https://github.com/jupyterlab/language-packs/\">language packs repository on GitHub</a>, and can be installed with pip. For example, it is possible to install the language pack for Simplified Chinese using the following command:</p><pre>pip install jupyterlab-language-pack-zh-CN</pre><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/0*8CYAXfBDt0v95iOc.png\" /><figcaption>JupyterLab in Simplified Chinese</figcaption></figure><p>Check out the <a href=\"https://jupyterlab.readthedocs.io/en/stable/user/language.html\">user documentation</a> to learn more about the feature and how to add new language packs.</p><h3>Improvements to the Simple Interface mode</h3><p>The Simple Interface mode (previously known as the Single-Document Mode) has been significantly updated to have a more streamlined, document-oriented feel.</p><p>To make the Simple Interface mode more discoverable, there is now a switch to toggle it in the status bar. You can also toggle it from the View menu or the command palette, or use the default keyboard shortcut Ctrl/Cmd+Shift+D.</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*ZLDBAmTHYyGVp3RNQN09KQ.gif\" /><figcaption>Enabling and disabling the Simple Interface Mode</figcaption></figure><p>Support for mobile has also greatly improved with a more compact layout. JupyterLab automatically switches to the Simple Interface when the window is resized down.</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*K24B_sxJ6Y-PHeLTURyRTw.gif\" /><figcaption>JupyterLab switches to the Simple Interface Mode on smaller screen</figcaption></figure><p>There is <a href=\"https://github.com/jupyterlab/jupyterlab/pull/9464\">ongoing work</a> to iterate on this concept and make the interface even more accessible on mobile devices.</p><h3>Install new extensions with pip and conda / mamba</h3><p>JupyterLab extensions can now be distributed as <em>prebuilt extensions</em>, which do not require a user to rebuild JupyterLab or have <a href=\"https://nodejs.org\">Node.js</a> installed. Prebuilt extensions can be distributed as Python packages using familiar package managers such as pip, conda, and mamba. This makes installing and using extensions much faster and more convenient!</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*DXwxXMj-B1Jqpsx3RmUitA.gif\" /><figcaption>Installing a new extension with pip</figcaption></figure><p>Prebuilt extensions can be published to <a href=\"https://pypi.org/\">PyPI</a> and <a href=\"https://conda-forge.org/\">conda-forge</a> as separate packages, or bundled into existing packages with Jupyter Server extensions and Classic Notebook extensions. This helps make the ecosystem as a whole more coherent.</p><p>For example, install the new ipywidgets 7.6.0 with pip or conda to automatically enable ipywidgets in both the classic Jupyter Notebook and in JupyterLab 3.0 — no extra installation step or JupyterLab rebuild needed!</p><figure><img alt=\"\" src=\"https://cdn-images-1.medium.com/max/1024/1*jg5drA_J783_IrqdLzye5w.gif\" /><figcaption>Installing ipywidgets works automagically in JupyterLab 3.0!</figcaption></figure><h3>Improved development workflow for Extension Authors</h3><p>The new prebuilt extensions are more convenient for extension authors to develop. The <a href=\"https://github.com/jupyterlab/extension-cookiecutter-ts\">TypeScript extension cookiecutter</a> has been updated to develop prebuilt extensions by default and provides all the necessary tooling to quickly create a new extension from scratch.</p><p>For more information, check out the <a href=\"https://jupyterlab.readthedocs.io/en/stable/extension/extension_dev.html\">Extension Developer Guide</a> and the <a href=\"https://jupyterlab.readthedocs.io/en/latest/extension/extension_migration.html\">JupyterLab 2.x to 3.x Migration Guide</a>.</p><h4>Extension Examples</h4><p>If you are looking for examples to learn how to make your own extension, make sure to check out the <a href=\"https://github.com/jupyterlab/extension-examples\">extension examples repository on GitHub.</a> The examples have been updated to be compatible with JupyterLab 3.0 and provide a hands-on approach to developing extensions.</p><h3>Changelog</h3><p>This post was only an overview of what’s new in 3.0!
